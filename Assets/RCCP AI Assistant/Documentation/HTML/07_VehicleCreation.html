The Vehicle Creation panel transforms raw 3D models into fully functional RCCP vehicles. This document covers the complete process from model requirements through configuration generation.

Prerequisites

Before creating a vehicle, ensure your 3D model meets these requirements:

Model Requirements

Separate Wheel Meshes

Each wheel must be a separate GameObject in the hierarchy, not baked into the body mesh. The assistant needs to identify and configure individual wheels.

Proper Scale

Your model should be at real-world scale. A typical passenger car is:

  • Length: 4-5 meters
  • Width: 1.7-2 meters
  • Height: 1.4-1.6 meters

Models significantly smaller or larger will trigger scale warnings.

Correct Orientation

RCCP expects:

  • Z-axis pointing forward (front of vehicle)
  • Y-axis pointing up (roof of vehicle)
  • X-axis pointing right (passenger side)

No Existing RCCP Components

The model should be a raw 3D model without existing RCCP components. If it already has RCCP_CarController, use the Vehicle Customization panel instead.

Scene Placement

The model must be:

  • In the scene (not just in Project window)
  • Selected in the Hierarchy
  • Active (not disabled)
  • At the root level or a clear root GameObject

Eligibility Checks

When you select a model, the assistant runs automatic eligibility checks:

Scale Check

Pass (Green): Dimensions are within typical vehicle ranges

Warning (Yellow): Unusual size detected (very small or very large)

Fail (Red): Scale is clearly wrong (sub-meter or enormous)

Common issues:

  • FBX imported at wrong scale (check import settings)
  • Model created at different scale in 3D software
  • Model is a component, not complete vehicle

Orientation Check

Pass: Z-forward, Y-up detected

Warning: May need rotation, but workable

Fail: Completely wrong orientation

The check analyzes the model's bounding box to determine facing direction.

Wheel Check

Pass: 4+ separate wheel meshes detected

Warning: Wheels detected but assignment uncertain

Fail: No separate wheels found (merged with body)

The assistant looks for:

  • GameObjects with "wheel" in name
  • Circular mesh shapes
  • Symmetric pairs in appropriate positions

Step-by-Step Process

Step 1: Select Your Model

  1. In Unity's Hierarchy window, select your 3D model's root GameObject
  2. The AI Assistant will show:
  • Model name and basic info
  • Mesh analysis with dimensions
  • Eligibility check results

Review the eligibility results:

  • Green checkmarks mean you're ready to proceed
  • Yellow warnings suggest reviewing but don't block creation
  • Red failures need to be addressed before continuing

Step 2: Analyze the Mesh Panel

Expand the mesh analysis to see detailed information:

Dimensions

Bounding box size in meters (Length x Width x Height)

Component Count

Number of child GameObjects, mesh renderers, etc.

Wheel Candidates

List of detected wheel transforms with:

  • Name and position
  • Estimated radius
  • Axle assignment guess (FL, FR, RL, RR)

Warnings

Any issues detected during analysis

Step 3: Write Your Description

In the prompt text area, describe the vehicle you want to create:

Include:

  • Vehicle type and category
  • Engine/power characteristics
  • Drivetrain configuration (FWD/RWD/AWD)
  • Handling characteristics
  • Intended use case

Example Descriptions:

Sports Car: 


High-performance sports car with a 3.0L turbocharged engine producing

400hp. Rear-wheel drive with a 6-speed manual transmission. Sharp,

responsive steering for track use. Firm suspension with minimal body

roll. Aggressive acceleration and high top speed.

Family SUV: 


Mid-size family SUV with a 2.5L naturally aspirated engine making

200hp. All-wheel drive with automatic transmission. Comfortable ride

quality with soft suspension for daily driving. Stable handling with

strong stability control. Good ground clearance for light off-road use.

Drift Car: 


Rear-wheel drive drift car with 350hp engine. Quick-ratio steering

for fast inputs. Minimal stability control for controlled slides.

Stiff suspension to maintain predictable weight transfer. High-rev

engine with accessible torque throughout the powerband.

Step 4: Generate Configuration

Click Generate to send your description to the AI.

During processing:

  • The button shows a loading state
  • A progress indicator appears
  • You cannot submit another request

The AI analyzes your description along with the mesh data and generates a complete vehicle configuration.

Step 5: Review Proposed Configuration

The response appears in a structured format showing:

Chassis Configuration

  • Mass and center of mass position
  • Drag coefficients
  • Physical properties

Engine Configuration

  • Power output
  • RPM ranges (idle, max)
  • Torque curves

Transmission

  • Gearbox type (automatic/manual)
  • Gear ratios
  • Shift behavior

Drivetrain

  • Differential type (FWD/RWD/AWD)
  • Torque distribution
  • Final drive ratio

Wheels and Suspension

  • Tire friction settings
  • Suspension spring/damper values
  • Travel distances

Stability Systems

  • ABS configuration
  • ESP/TCS settings
  • Steering assistance

Step 6: Apply the Configuration

Review the proposed changes, then choose:

Apply All

Creates the vehicle with all proposed settings.

Apply Selected

Use toggles to enable/disable component groups before applying.

Discard

Clear the response without creating the vehicle.

When you click Apply:

  1. RCCP_CarController is added to the root
  2. All required RCCP components are configured
  3. WheelColliders are created for each wheel
  4. The vehicle enters post-creation state

What the AI Configures

The creation process sets up these RCCP components:

RCCP_EnginePower, RPM, torque curves

Handling Eligibility Issues

Scale Problems

If your model is incorrectly scaled:

  1. Check the FBX import settings
  2. Verify "Scale Factor" is appropriate
  3. Consider the source application's export settings
  4. Rescale in Unity if necessary (affects children)

Orientation Problems

If orientation is wrong:

  1. Create an empty parent GameObject
  2. Place your model as a child
  3. Rotate the child to correct orientation
  4. Select the parent for vehicle creation

Missing Wheels

If wheels aren't detected:

  1. Ensure wheels are separate GameObjects
  2. Include "wheel" in wheel names
  3. Check that wheels aren't merged into body mesh
  4. Separate meshes in your 3D software if needed

Tips for Best Results

Be Specific About Performance

Instead of "fast" say "400hp, 0-60 in 4 seconds."

Instead of "handles well" say "responsive steering with minimal body roll."

Reference Real Vehicles

"Similar to a Porsche 911" gives the AI context about expected behavior.

Describe the Use Case

"Track day car" vs "daily driver" implies different configurations.

Start Conservative

You can always add more power or loosen stability later. Starting too extreme makes tuning harder.

Use the Right Model

Ensure your 3D model matches what you're describing. A truck model described as a sports car won't produce good results.

After Creation

After applying the configuration, the assistant enters post-creation mode with wizards for:

  • Wheel assignment (confirm which wheels are which)
  • Body collider setup (add physics colliders)

See the Post-Creation Setup documentation for details on these steps.

Common Questions

Can I undo vehicle creation?

Yes, use Edit > Undo (Ctrl+Z) immediately after creation to revert.

What if the AI misunderstands my description?

Discard the response and try a more specific description. Iterate until you get the right configuration.

Can I modify the configuration after creation?

Yes, use the Vehicle Customization panel or any specialized panel (Behaviors, Wheels, etc.) to modify existing vehicles.

Do I need to set up physics manually?

The creation process handles basic physics setup. You may want to add colliders using the Body Colliders wizard for complete physics.
