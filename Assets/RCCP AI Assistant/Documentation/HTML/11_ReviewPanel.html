The Review Panel provides a structured preview of AI-generated configurations before applying them. This allows you to understand exactly what changes will be made and selectively apply only the modifications you want.

Purpose

The Review Panel serves several important functions:

Transparency

See exactly what the AI is proposing before any changes are made to your vehicle.

Control

Enable or disable specific component groups to apply only desired changes.

Comparison

View before and after values side-by-side to understand the magnitude of changes.

Safety

Review potentially impactful changes before committing them to your vehicle.

Structured Change Preview

How Changes Are Displayed

When the AI generates a response, the Review Panel organizes changes into a hierarchical structure: 


Engine

  ├─ maxPower: 250 → 350 hp

  ├─ maxRPM: 6500 → 7500

  └─ torqueCurve: [modified]



Gearbox

  ├─ gearCount: 5 → 6

  └─ gearRatios: [modified]



Stability

  ├─ ABS: disabled → enabled

  ├─ ABSStrength: 0.0 → 0.7

  └─ ESP: disabled → enabled

Component Sections

Changes are grouped by RCCP component:

SectionComponents Covered
ChassisMass, center of mass, drag
EnginePower, RPM, torque curves
GearboxGear ratios, shift timing
DifferentialDrive type, torque split
WheelsSuspension, friction, alignment
StabilityABS, ESP, TCS settings
AudioEngine sound configuration
LightsLight setup
DamageDeformation, detachable parts

Collapsible Groups

Each component section can be collapsed or expanded:

  • Click the section header to toggle
  • Collapsed sections show a summary (e.g., "3 changes")
  • Expand to see individual property changes

Before/After Comparison

For each modified property, you see:

Property Name

The RCCP component field being changed.

Before Value

The current value on your vehicle (captured when you clicked Generate).

After Value

The proposed new value from the AI.

Change Indicator

Visual indication of change type:

  • Arrow (→) for value changes
  • Plus (+) for newly enabled features
  • Minus (-) for disabled features

Complex Value Display

For complex values like arrays or curves:

  • Simple arrays show element count
  • Curves show "modified" with option to expand
  • Nested objects show child changes indented

Selective Apply

Toggle Controls

Each component group has a toggle switch:

  • Enabled (blue): This group's changes will be applied
  • Disabled (gray): This group's changes will be skipped

By default, all groups are enabled after generation.

Selective Workflow

  1. Review all proposed changes
  2. Disable groups you don't want
  3. Click Apply Selected
  4. Only enabled groups are applied

Use Cases for Selective Apply

Partial Agreement

The AI's engine changes are good, but the stability changes are too aggressive. Disable the Stability group.

Incremental Application

Apply changes in stages to test each component's effect individually.

Preserving Manual Work

You manually tuned the suspension. Disable the Wheels group to preserve your work.

Testing Individual Components

Apply only one group at a time to isolate its effect on vehicle behavior.

Actions Available

Apply All

Applies every change in the response, regardless of toggle states.

When to use:

  • You've reviewed everything and approve all changes
  • Quick application when you trust the AI's judgment
  • Time-sensitive prototyping

Apply Selected

Applies only the changes from enabled (toggled on) component groups.

When to use:

  • You want partial changes
  • Testing specific modifications
  • Preserving some existing configuration

Discard

Clears the response without making any changes.

When to use:

  • The response doesn't match your intent
  • You want to try a different prompt
  • Canceling the operation

Copy JSON to Clipboard

Copies the raw JSON response to your clipboard.

When to use:

  • Saving the configuration for later
  • Sharing with team members
  • Debugging or documentation
  • Manual editing before applying

View Raw Response

Toggle between structured view and raw JSON display.

When to use:

  • Debugging AI responses
  • Understanding the exact data format
  • Checking for parsing issues

Reading the Review Panel Effectively

Color Coding

The panel uses colors to convey information:

  • Blue: Primary accent, enabled states
  • Green: Value increases, additions
  • Red: Value decreases, removals
  • Gray: Unchanged, disabled states

Numeric Changes

Pay attention to magnitude:

  • Small changes (5-10%): Fine-tuning
  • Medium changes (20-50%): Noticeable impact
  • Large changes (100%+): Significant shift in behavior

Boolean Changes

Watch for enable/disable toggles:

  • "disabled → enabled": Feature activation
  • "enabled → disabled": Feature deactivation

Array/Curve Changes

For complex data:

  • Gear ratios: Check each gear's ratio
  • Torque curves: May need testing to feel the change
  • Light positions: Review in Scene view after applying

Best Practices

Always Review Before Applying

Even if you trust the AI, a quick scan can catch:

  • Misunderstood intent
  • Unexpected side effects
  • Extreme values

Use Selective Apply for Uncertainty

When unsure about a component:

  1. Disable uncertain groups
  2. Apply confident changes
  3. Test the vehicle
  4. Generate new prompt for remaining adjustments

Compare Magnitudes

Large changes deserve extra scrutiny:

  • A 50% power increase is significant
  • A 200% change might be an AI error
  • Very small changes (<1%) might not be worth applying

Check Related Components

Some changes interact:

  • Power increase might need gear ratio adjustment
  • Suspension changes affect handling balance
  • Stability system changes affect driving feel

Save Before Major Changes

Unity's Undo works, but for safety:

  • Save the scene before applying large changes
  • Consider making a prefab backup of the vehicle

Integration with Workflow

In Vehicle Creation

After generating a new vehicle configuration:

  1. Review the complete setup
  2. Disable any components you want to configure manually
  3. Apply to create the vehicle
  4. Proceed to post-creation wizards

In Vehicle Customization

After generating modifications:

  1. Review against your current vehicle state
  2. Identify unexpected or unwanted changes
  3. Apply selectively or all at once
  4. Test immediately to validate

With Specialized Panels

Each specialized panel (Behaviors, Wheels, etc.):

  1. Shows domain-specific changes only
  2. Uses the same review structure
  3. Offers the same selective apply controls

Troubleshooting

"No changes detected"

The AI response didn't include modifications. Try:

  • A more specific prompt
  • Confirming the vehicle was selected
  • Checking if desired changes are possible

Values Look Wrong

If proposed values seem extreme:

  • The AI may have misunderstood units
  • Your prompt may have been ambiguous
  • Try a more explicit description

Some Components Missing

Not all components appear in every response:

  • The AI only shows changed values
  • Unchanged components don't appear
  • Request specific components explicitly if needed

Toggle States Not Saving

Toggle states reset after each generation:

  • Intentional behavior for clean starts
  • Manually set toggles each time
  • Or use "Apply All" for quick application
