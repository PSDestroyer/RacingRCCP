Window Modes

The assistant offers two window modes, configurable in Settings:

Utility Mode (Default)

  • Fixed size window (800x850 pixels)
  • Floats above other windows
  • Cannot be docked to the editor layout
  • Best for: Quick access, focused work

Dockable Mode

  • Resizable window
  • Can be docked alongside other panels
  • Integrates with your editor layout
  • Best for: Permanent placement, large monitors

To switch modes:

  1. Open Settings
  2. Find UI Settings section
  3. Toggle "Use Utility Window"
  4. Click "Restart Window" to apply

Sidebar

The sidebar on the left provides navigation and quick access to features.

Additional Sidebar Items

Below the panel tabs:

Prompt History

Access your complete prompt history. Search, filter, and re-apply previous configurations.

Settings

Open the settings panel to configure API access, UI preferences, and developer options.

Context Bar

The context bar appears at the top of the main area, providing essential information at a glance.

Vehicle Selection Display

Shows your currently selected vehicle:

  • Name: GameObject name from hierarchy
  • Type: "Raw Model" or "RCCP Vehicle"
  • Status indicators: Warnings or issues

Usage Display

Shows your current API usage status:

  • Setup Phase: "Setup: X/100 remaining"
  • Free Phase: "Daily: X/10 remaining | Resets in Xh Xm"
  • Own Key: "Using your API key (unlimited)"

Model Indicator

When relevant, shows which Claude model will be used for the next request.

Main Workflow Area

The central area displays the three-step workflow for the current panel.

Step 1: Select Target

This step shows information about your selection:

For Vehicle Creation:

  • Mesh analysis summary
  • Eligibility checks (scale, orientation, wheels)
  • Size warnings if applicable
  • Multi-selection count for batch operations

For Customization Panels:

  • Current vehicle information
  • Component presence indicators
  • Current configuration snapshot

Step 2: Describe What You Want

The input area for your natural language prompt:

Mode Toggle (Configure / Ask)

At the top of the input area, two buttons let you switch interaction modes:

ModeButton ColorPurpose
ConfigureOrangeAI returns settings to review and apply
AskBlueAI returns explanations (no changes)

A hint below the buttons confirms the current mode:

  • Configure: "Returns settings you can review and apply"
  • Ask: "Returns advice only (no changes)"

Use Configure mode for making changes, Ask mode for questions. See the Ask Mode documentation for details.

Text Input Field

A multi-line text area where you type your description. Supports:

  • Multiple lines (Shift+Enter for new lines when Enter sends)
  • Up to 1000 characters (configurable)

Quick Prompts

Clickable buttons with common requests. Each panel has its own set of quick prompts relevant to that panel's purpose.

Character Counter

Shows current length vs maximum allowed.

Generate Button

Click to send your prompt to the AI. The button text changes based on mode:

  • Configure mode: "Generate"
  • Ask mode: "Ask"

Disabled when:

  • No text entered
  • Invalid selection
  • Currently processing
  • Rate limit reached

Step 3: Review and Apply

After generating, this step shows the proposed changes.

Note: In Ask mode, Step 3 displays a text response instead of the structured review panel. There are no apply buttons because Ask mode doesn't make changes. See the Ask Mode documentation for details.

Configure Mode Response:

Change Summary

A structured view of proposed modifications organized by component:

  • Component group headers (collapsible)
  • Property names with before/after values
  • Visual indicators for changes

Selective Apply Toggles

Toggle switches for each component group. Enable/disable which groups to apply.

Action Buttons

  • Apply All: Apply everything in the response
  • Apply Selected: Apply only toggled groups
  • Discard: Clear the response without applying

Additional Actions

  • Copy JSON to clipboard
  • View raw response (developer mode)

Ask Mode Response:

In Ask mode, the response area shows a plain text explanation instead of structured changes. A "Clear" button dismisses the response when you're done reading.

Status Bar

The bottom bar shows:

Status Messages

Feedback about the last operation:

  • Success messages (green)
  • Warnings (yellow)
  • Errors (red)
  • Processing status

Branding

"RCCP AI Assistant" identifier.

Messages auto-fade after a few seconds to reduce visual clutter.

Keyboard Shortcuts

When enabled in settings:

ShortcutAction
EnterGenerate (when sendOnEnter enabled)
Ctrl+EnterGenerate (when sendOnEnter disabled)
Shift+EnterNew line in text field

Panel-Specific UI Elements

Different panels may show additional UI elements:

Vehicle Creation

  • Eligibility status with expand/collapse
  • Wheel candidate visualization
  • Post-creation wizard (after applying)

Lights Panel

  • Vision detection button
  • Light preview with position markers

Damage Panel

  • Part detection visualization
  • Strength/mass indicators

Diagnostics Panel

  • Run Diagnostics button (replaces Generate)
  • Results list with severity filters
  • No AI response area (local only)

Animations

The UI includes smooth animations for:

  • Panel transitions when switching tabs
  • Response appearance when AI returns
  • Status message fade-out
  • Success flash on apply
  • Processing pulse indicator

Animations can be disabled or speed-adjusted in Settings > Animation Settings.

Visual Indicators

Colors

The UI uses consistent colors for status:

  • Blue/Cyan: Primary accent, active selections
  • Green: Success, valid, enabled
  • Yellow/Orange: Warning, caution
  • Red: Error, invalid, disabled

Status Icons

IconMeaning
CheckValid, passed, applied
WarningPotential issue, review needed
X/CrossInvalid, failed, error
SpinnerProcessing, loading

Progress Indicators

  • Linear progress bars for usage limits
  • Pulsing effect during AI requests
  • Animated transitions between states

Responsive Behavior

The window adapts to different scenarios:

No Selection

Step 1 prompts you to select a vehicle or model.

Invalid Selection

Shows what's wrong and what to select instead.

Processing

Disables inputs, shows progress indicator, prevents duplicate requests.

Error State

Shows error message with recovery suggestions.

Tips for Effective Use

Organize Your Layout (Dockable Mode)

Dock the AI Assistant next to your Scene view for easy selection and configuration.

Use Keyboard Navigation

Learn the shortcuts for faster workflow. Enable sendOnEnter for rapid iteration.

Expand/Collapse Strategically

Keep frequently-used sections expanded, collapse others to reduce scrolling.

Watch the Context Bar

The context bar gives instant feedback on selection validity and usage status.

Check Status Messages

Status messages provide important feedback - don't dismiss them too quickly.
