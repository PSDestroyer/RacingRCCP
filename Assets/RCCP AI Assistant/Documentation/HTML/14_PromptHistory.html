Prompt History provides a global log of all prompts and responses across all vehicles and sessions. Unlike Vehicle History which is per-vehicle, Prompt History gives you a searchable archive of your AI interactions.

What Is Prompt History

Global History

Prompt History tracks all AI interactions:

  • Every prompt you've sent
  • Every response you've received
  • Across all vehicles
  • Across all panels
  • Persists between Unity sessions

What Gets Recorded

Each entry includes:

  • Timestamp
  • Panel type used
  • Vehicle name (at time of prompt)
  • User prompt text
  • AI response (JSON)
  • Whether it was applied
  • Whether it was informational (question vs configuration)

Storage

Prompt History is stored in:

  • Unity's EditorPrefs
  • Persists across sessions
  • Not tied to specific scenes or vehicles
  • Available immediately on opening the assistant

Accessing Prompt History

Opening the Panel

  1. In the AI Assistant sidebar, click Prompt History
  2. The history panel opens
  3. Recent entries appear at the top

Searching and Filtering

Search

Type in the search box to filter entries:

  • Searches prompt text
  • Searches vehicle names
  • Searches response content
  • Real-time filtering

Filters

Filter by various criteria:

By Applied Status:

  • All entries
  • Applied only
  • Not applied only

By Panel Type:

  • All panels
  • Vehicle Creation
  • Vehicle Customization
  • Behaviors
  • Wheels
  • Audio
  • Lights
  • Damage

By Date:

  • All time
  • Today
  • This week
  • This month

Sorting

Change the sort order:

  • Newest first (default)
  • Oldest first
  • By vehicle name
  • By panel type

Entry Details

Viewing an Entry

Click an entry to expand it and see:

Full Prompt

The complete text you wrote.

Full Response

The complete AI response, formatted for readability.

Metadata

  • Exact timestamp
  • Panel used
  • Vehicle name
  • Applied status

Collapsible JSON

The response JSON can be:

  • Collapsed for overview
  • Expanded for full detail
  • Organized by component section

Actions

Copy JSON

Copy the AI response to clipboard:

  • Click Copy JSON on any entry
  • Paste elsewhere as needed
  • Useful for manual application or documentation

Re-Apply to Current Vehicle

Apply a past configuration to the currently selected vehicle:

  1. Select an RCCP vehicle in the scene
  2. Open Prompt History
  3. Find the entry you want to reuse
  4. Click Re-Apply
  5. The configuration is applied to the current vehicle

Note: This applies the stored response directly, without regenerating from the AI.

Delete Entry

Remove entries you no longer need:

  • Click Delete on individual entries
  • Or use bulk selection to delete multiple

Bulk Selection

Enable bulk mode to select multiple entries:

  1. Click Bulk Select toggle
  2. Check entries to select
  3. Apply bulk actions:
  • Delete selected
  • Export selected
  • Copy selected

Per-Entry Information

Timestamp

When the prompt was sent:

  • Full date and time
  • Relative time ("3 days ago")

Panel Type

Which panel was used:

  • Helps filter by task type
  • Indicates what kind of configuration

Vehicle Name

The vehicle that was selected:

  • May no longer exist in current scene
  • Historical reference

User Prompt

Your original prompt:

  • Preserved exactly as written
  • Learn from past successful prompts

AI Response

The JSON response:

  • Complete configuration data
  • Can be re-applied

Applied Status

Whether you applied this response:

  • Applied: Changes were committed
  • Not Applied: Response was discarded

Informational Flag

Whether this was a question vs configuration:

  • Informational responses don't have JSON config
  • Questions like "explain how ABS works"

Use Cases

Finding Successful Prompts

When you need a similar configuration:

  1. Search for relevant keywords
  2. Find entries that were applied
  3. Review the prompts that worked
  4. Adapt for your current need

Reusing Configurations

To apply a past config to a new vehicle:

  1. Open Prompt History
  2. Find the desired configuration
  3. Select your new vehicle
  4. Re-apply the entry

Learning What Works

Study your history to understand:

  • Which prompts produce good results
  • What level of detail is needed
  • How to phrase requests effectively

Configuration Documentation

Use history as documentation:

  • Record of all changes made
  • Searchable by vehicle or feature
  • Exportable for reference

Recovering Lost Configurations

If you need to recreate a vehicle:

  1. Find the original creation entry
  2. Create a new vehicle from the same model
  3. Re-apply the stored configuration

Best Practices

Write Descriptive Prompts

Prompts appear in history - make them clear:

  • Describe what you want clearly
  • Include context when helpful
  • Future searches will thank you

Apply Good Results

When a configuration works well:

  • Actually apply it (not just preview)
  • "Applied" status helps find successes later

Clean Up Failed Experiments

Periodically remove:

  • Failed attempts
  • Duplicate prompts
  • Outdated configurations

Use Search Effectively

Instead of scrolling:

  • Search for vehicle names
  • Search for feature keywords
  • Search for panel-specific terms

Export Important Configurations

For critical configurations:

  • Copy JSON to a text file
  • Store outside Unity as backup
  • Document with the prompt used

Integration with Vehicle History

Different Purposes

Prompt History:

  • Global across all vehicles
  • Focus on prompts and responses
  • For finding and reusing configurations
  • Stored in EditorPrefs

Vehicle History:

  • Specific to one vehicle
  • Includes before/after states
  • For restore and comparison
  • Stored in scene

Using Together

  1. Use Prompt History to find successful prompts
  2. Re-apply them to new vehicles
  3. Vehicle History tracks the application on each vehicle
  4. Use Vehicle History to restore if needed

Cross-Referencing

A single prompt in Prompt History may have been applied to multiple vehicles. Each application creates a separate Vehicle History entry on each vehicle.

Troubleshooting

History Not Saving

If entries disappear:

  • EditorPrefs may have been cleared
  • Different Unity installation
  • Check EditorPrefs storage

Can't Find an Entry

If you can't find a past entry:

  • Check filters aren't hiding it
  • Search for different keywords
  • Entry may have been deleted

Re-Apply Not Working

If re-apply fails:

  • Ensure a valid RCCP vehicle is selected
  • Check the stored response is valid JSON
  • Verify the vehicle can receive the configuration

Entry Shows Wrong Vehicle

The vehicle name is captured at prompt time:

  • If vehicle was renamed, old name shows
  • If vehicle was deleted, name is still recorded
  • This is expected behavior
