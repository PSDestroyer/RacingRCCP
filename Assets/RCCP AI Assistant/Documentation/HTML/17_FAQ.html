Cost and Pricing

How much does RCCP AI Assistant cost to use?

The assistant itself is included with your purchase. For AI functionality:

Server Proxy (Free Tier):

  • 400 setup requests (no daily limit) + 20 per hour burst for initial evaluation
  • After that: 20 requests per day + 10 per hour burst
  • No additional cost

Own API Key:

  • Pay Anthropic directly per request
  • Typical vehicle configuration: $0.01-0.30 depending on model
  • Haiku is cheapest, Opus is most expensive

Which AI model should I use?

Haiku 4.5: Best for quick iterations and simple configurations. Very fast and cheap.

Sonnet 4.5: Best for most use cases. Good balance of quality, speed, and cost.

Opus 4.5: Best for complex requests where quality matters most. Slowest and most expensive.

Start with Sonnet for normal use. Use Haiku for rapid prototyping. Use Opus for important final configurations.

Can I use it without paying for API access?

Yes, the server proxy provides free tier access:

  • 100 initial requests for setup and evaluation
  • 10 requests per day after that
  • No credit card or API key required

Features and Capabilities

Can I ask the AI questions instead of making changes?

Yes, use Ask Mode. At the top of the prompt area, click the "Ask" button (turns blue) instead of "Configure" (orange). In Ask mode:

  • The AI returns plain text explanations
  • No configuration changes are proposed
  • No review panel or apply buttons appear
  • Great for learning about vehicle physics

Example questions:

  • "What does the differential torque split mean?"
  • "What power level is appropriate for a family sedan?"
  • "Why might my car understeer in corners?"

See the Ask Mode documentation for details.

What's the difference between Configure and Ask mode?

AspectConfigure ModeAsk Mode
PurposeMake changesGet answers
ResponseJSON settingsPlain text
Apply buttonsYesNo
Affects vehicleYes (if applied)Never

Both modes use your API quota. Ask mode is for learning; Configure mode is for making changes.

Can I use it offline?

No, an internet connection is required. The assistant needs to communicate with either:

  • The Anthropic Claude API (own key mode)
  • The BoneCracker Games server proxy

Offline functionality is not available because the AI processing happens remotely.

Does it work with multiple Unity projects?

Yes. If using the server proxy:

  • Your device token persists in EditorPrefs
  • Opening different projects uses the same quota
  • Usage is per-device, not per-project

If using your own API key:

  • Key is stored per-user in EditorPrefs
  • Works in any project on that machine

What happens when I update RCCP?

After updating RCCP:

  1. Go to Tools > BoneCracker Games > RCCP AI Assistant > Extract Component Defaults
  2. This syncs the AI's knowledge with current RCCP default values
  3. Run this after each RCCP update

The assistant should continue working, but extracting defaults ensures optimal configurations.

Can I customize the AI prompts?

Yes, the system uses ScriptableObject prompt assets:

  • Located in Assets/RCCP AI Assistant/Resources/Prompts/
  • Each panel has its own prompt asset
  • Advanced users can modify system prompts

Warning: Incorrect prompt modifications can produce poor results. Make backups before editing.

Data and Privacy

Is my data sent to the server?

What IS sent:

  • Your prompt text
  • Vehicle configuration requests

What is NOT sent:

  • Your vehicle's geometry or mesh data
  • Scene information
  • Project files
  • Personal information beyond device fingerprint

The server proxies requests to Claude and tracks usage. No vehicle data is stored permanently.

Is my API key stored securely?

Your API key is stored in Unity's EditorPrefs:

  • Local to your machine
  • Not transmitted to BoneCracker Games servers
  • Only sent to Anthropic for authentication

Keep your key private. Don't share or commit it to version control.

Can others see my configurations?

No. Your prompts and configurations are:

  • Not stored permanently on servers (server proxy mode)
  • Between you and Anthropic (own key mode)
  • Local to your machine (history features)

Usage Limits

How do I get more requests?

Options to increase your request capacity:

  1. Wait for reset - Daily limit resets at midnight
  2. Use your own API key - Unlimited requests (pay per use)
  3. Upgrade to premium - Contact BoneCracker Games for enterprise options

Do unused requests carry over?

No. Daily allowances reset at midnight regardless of usage. Use them or lose them.

What counts as a request?

One request = one click of the Generate button that receives a response.

Counts as a request:

  • Successful generation with response
  • Failed generation that reached the AI

Does NOT count:

  • Validation errors before sending
  • Running diagnostics (local only)
  • Viewing history or settings

Technical Questions

Why do I need Editor Coroutines?

Unity's standard coroutines only work in Play mode. Editor Coroutines allows the assistant to make asynchronous API calls while in Edit mode, enabling a seamless workflow without entering Play mode.

Why Unity 6 only?

RCCP AI Assistant uses features and APIs introduced in Unity 6:

  • Specific editor API improvements
  • Serialization features
  • UI toolkit capabilities

Backporting to earlier Unity versions is not planned.

Can I use it with prefabs?

Best practices for prefabs:

Creation:

  • Don't convert a prefab directly
  • Create vehicle from scene instance
  • Make it a prefab after configuration

Customization:

  • Unpack prefab instance before modifying
  • Or open prefab in isolation mode
  • Apply changes then save prefab

RCCP components have specific prefab behavior that works best with proper prefab workflows.

Workflow Questions

Should I create or customize?

Use Vehicle Creation when:

  • Starting from a raw 3D model
  • Model has no RCCP components
  • Building a new vehicle from scratch

Use Vehicle Customization when:

  • Vehicle already has RCCP_CarController
  • Modifying an existing setup
  • Adjusting a vehicle you created earlier

What's the difference between panels?

PanelUse For
Vehicle CreationConverting raw models to RCCP vehicles
Vehicle CustomizationGeneral modifications to existing vehicles
BehaviorsStability systems, steering, driving feel
WheelsTires, suspension, alignment
AudioEngine sound configuration
LightsVehicle lighting setup
DamageDeformation, breakable parts
DiagnosticsValidation (no AI)

Specialized panels have focused AI context for better results in their domain.

How specific should my prompts be?

More specific is generally better:

Too vague:

"Make it faster"

Better:

"Increase engine power to 350hp with improved torque at low RPM for better acceleration"

Include:

  • Specific values when you know them
  • Behavior descriptions when you don't
  • Context about intended use

Can I undo AI changes?

Yes, several ways:

  1. Unity Undo (Ctrl+Z) - Immediately after applying
  2. Vehicle History - Restore to previous state
  3. Don't Apply - Preview changes without applying

Can the Diagnostics panel fix issues automatically?

Yes, many common issues can be auto-fixed:

  1. Run diagnostics on your vehicle
  2. If fixable issues are found, an Auto-Fix button appears
  3. Click it to automatically repair what's possible
  4. Diagnostics run again to verify

Auto-fix can resolve: broken references, invalid values, and missing component setup. It cannot fix structural problems like merged wheel meshes or incorrect model scale. All auto-fixes support Undo.

Common Problems

Why won't my model work for vehicle creation?

Most common issues:

  1. Wheels merged with body - Separate them in 3D software
  2. Wrong scale - Check import scale factor
  3. Wrong orientation - Z should be forward, Y should be up
  4. Not in scene - Select from Hierarchy, not Project

Why does my vehicle flip over?

Common causes:

  1. Center of mass too high - Lower it in RCCP_CarController
  2. Missing colliders - Add body colliders
  3. Wheel collider issues - Check wheel setup

Why does my vehicle not move?

Check these:

  1. Run Diagnostics for errors
  2. Verify drivetrain is configured
  3. Check input system is set up
  4. Ensure vehicle is active
  5. Test in Play mode, not Edit mode

  3. Note your Unity version and RCCP version

  5. Or email support with your documentation

  • Additional panel types
  • Enhanced prompt templates
  • Performance optimizations
  • User-requested features

Check release notes and announcements for updates.
