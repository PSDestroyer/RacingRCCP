The Lights panel configures vehicle lighting including headlights, taillights, brake lights, reverse lights, and indicators. A unique feature of this panel is vision-based light detection, which uses AI to automatically identify light positions from your vehicle's appearance.

What This Panel Configures

Light Types

RCCP vehicles support multiple light types:

Headlights

Front-facing lights for illumination:

  • Low beam and high beam
  • Intensity and range settings
  • Color configuration (typically white)

Taillights

Rear-facing lights for visibility:

  • Always on when headlights are active
  • Lower intensity than headlights
  • Red color

Brake Lights

Activated when braking:

  • Higher intensity than taillights
  • Same or different light sources
  • Red color

Reverse Lights

Activated when in reverse gear:

  • White color
  • Illuminates behind vehicle
  • Lower intensity than headlights

Indicators (Turn Signals)

Directional signals for turning:

  • Left and right pairs
  • Front and rear positions
  • Amber/orange color
  • Blinking behavior

Light Properties

Each light has configurable properties:

Intensity

How bright the light appears:

  • Higher values = brighter
  • Affects both visual and illumination

Range

How far the light reaches:

  • Important for headlights
  • Affects actual scene illumination

Spot Angle

The cone width for spot lights:

  • Narrow = focused beam
  • Wide = spread illumination

Color

The light's color:

  • Headlights: White (or slight tint)
  • Taillights/Brake: Red
  • Indicators: Amber/Orange
  • Reverse: White

Lens Flares

Visual effects for light sources:

Flare Intensity

How prominent the flare effect is.

Flare Size

How large the flare appears.

Enable/Disable

Toggle flares per light or globally.

Breakable Lights

Integration with the damage system:

Breakable

Whether this light can be damaged.

Break Force

How much impact force breaks the light.

Broken State

Light turns off when broken.

Vision-Based Light Detection

What It Is

The Lights panel includes a unique AI feature that analyzes screenshots of your vehicle to automatically detect light positions. Instead of manually placing each light, the AI identifies where headlights, taillights, and other lights should go based on the vehicle's visual appearance.

How It Works

  1. Click Detect Lights button
  2. The assistant captures screenshots of your vehicle from multiple angles
  3. Screenshots are sent to Claude's vision model
  4. The AI analyzes the images and identifies light locations
  5. Detected positions are returned as world coordinates
  6. Preview shows proposed light positions on the vehicle

What Gets Detected

The vision system can identify:

  • Headlight positions (left and right)
  • Taillight positions (left and right)
  • Additional lights if visible
  • Approximate light shapes and directions

Using Vision Detection

  1. Select your RCCP vehicle in the scene
  2. Ensure the vehicle is visible (not occluded)
  3. Click Detect Lights
  4. Wait for processing (uses vision model)
  5. Review the preview with light markers
  6. Adjust positions if needed
  7. Apply the configuration

Cost Considerations

Vision-based detection uses Claude's vision model:

  • Approximately 10x the cost of text requests
  • Uses one of your daily requests (if on free tier)
  • Consider using for initial setup, then manual adjustment

Preview and Adjustment

After detection, a preview shows:

  • Markers at detected light positions
  • Color coding by light type
  • Handles for manual position adjustment

You can:

  • Accept all positions
  • Adjust individual lights
  • Re-detect with different angles
  • Discard and configure manually

Batch Light Detection

The Lights panel supports batch operations for configuring multiple vehicles at once.

When to Use Batch Mode

Batch light detection is useful when:

  • Setting up a fleet of similar vehicles
  • Applying consistent lighting across vehicle variants
  • Initial setup of multiple vehicles in a project
  • Standardizing light configurations across a roster

How to Select Multiple Vehicles

  1. In the Hierarchy window, hold Ctrl (Windows) or Cmd (Mac)
  2. Click each RCCP vehicle you want to include
  3. Or use Shift+Click to select a range of vehicles

The Lights panel shows:

  • "X vehicles selected" indicator
  • List of selected vehicle names
  • Batch mode UI

Batch Detection Workflow

  1. Select multiple RCCP vehicles in the scene
  2. Switch to the Lights panel
  3. Write a description of the lighting setup you want
  4. Click Generate to create the configuration
  5. The same lighting configuration applies to all vehicles
  6. Review the proposed changes
  7. Click Apply to configure all vehicles

How Batch Apply Works

When you apply in batch mode:

  1. Each vehicle is processed sequentially
  2. Light positions are calculated relative to each vehicle's bounds
  3. The same light types and properties are applied
  4. Progress is shown during processing
  5. A summary reports successes and any failures

Vision Detection in Batch Mode

If using vision-based detection with multiple vehicles:

  • Each vehicle is screenshotted individually
  • Light positions are detected per-vehicle
  • Results may vary based on each vehicle's appearance
  • Processing takes longer with more vehicles

Tip: For consistent results across similar vehicles, consider detecting lights on one vehicle, then using a text prompt to apply the same configuration to others.

Batch Limitations

Same Configuration

All vehicles receive the same light types and properties. Individual customization requires single-vehicle mode.

Processing Time

Batch operations take longer. Allow time for all vehicles to process.

Vision Cost

Vision detection for each vehicle uses API quota. Consider using text prompts for batch operations to reduce costs.

Undo Support

Batch light operations create a single undo group. One Ctrl+Z reverts all vehicles to their previous state.

Manual Light Configuration

Without Vision Detection

You can configure lights manually through prompts: 


Set up headlights at the front corners of the vehicle.

Position taillights at the rear lower corners.

Add brake lights above the taillights.

Configure indicators at all four corners.

The AI will generate reasonable positions based on typical vehicle layouts and your description.

Refining Detected Positions

After vision detection, refine with prompts: 


Move the right headlight 10cm to the left.

Raise the taillights by 5cm.

Increase headlight intensity to 2.0.

Complete Manual Setup

For full control: 


Create headlights:

- Position: front of vehicle, 40cm from center on each side

- Height: 70cm from ground

- Intensity: 2.0, range: 50m

- Spot angle: 60 degrees, white color



Create taillights:

- Position: rear of vehicle, 50cm from center on each side

- Height: 60cm from ground

- Intensity: 1.0, range: 10m

- Red color

Common Configurations

Standard Passenger Car 


Configure standard automotive lighting:

- Dual headlights with high/low beam

- Combined taillights and brake lights

- Reverse lights below taillights

- Indicators at all four corners

- Enable lens flares for realism

- Make all lights breakable for damage integration

Sports Car 


Set up aggressive sports car lighting:

- Thin, wide headlights with high intensity

- LED-style taillights

- Strong brake lights

- Flush-mounted indicators

- Minimal flares for modern look

Classic Car 


Configure classic car lighting:

- Round headlights with visible bulb

- Small round taillights

- Separate brake lights

- Large visible indicators

- Prominent lens flares

Emergency Vehicle 


Set up emergency vehicle lighting:

- Standard headlights and taillights

- Add roof-mounted light bar

- Alternating flash pattern for roof lights

- High intensity for all warning lights

Damage Integration

Breakable Lights

When integrated with the damage system: 


Make headlights breakable with 500N impact force.

Taillights should break at 300N.

Broken lights turn off and show damage material.

Damage Response

Configure how lights respond to damage:

  • Break force threshold
  • Visual change when broken
  • Whether they can be repaired

Prompt Examples

Brightness Adjustment 


Increase headlight brightness for better night visibility.

Set intensity to 2.5 with range of 50 meters.

Color Customization 


Add a slight blue tint to the headlights for modern look.

Keep taillights pure red.

Flare Configuration 


Enable subtle lens flares on all lights.

Headlight flares should be larger than taillight flares.

Position Refinement 


The left headlight is too far inward.

Move it 15cm toward the left edge of the vehicle.

Tips for This Panel

Use Vision Detection First

For new vehicles, let the AI detect initial positions, then refine manually.

Test at Night

Switch to a dark scene to properly evaluate light appearance and intensity.

Match the Vehicle Style

Classic cars have different light styles than modern cars. Match the era.

Consider Performance

Too many lights or very high ranges can impact performance. Balance as needed.

Integrate with Damage

For full realism, configure light damage alongside the Damage panel settings.

Preview Before Applying

Use the position preview to verify lights are where you want them before committing.
