After creating a vehicle with the AI Assistant, additional setup steps ensure your vehicle is fully functional. This document covers the wheel assignment wizard, body colliders wizard, and transitioning to customization.

Post-Creation State

When vehicle creation completes successfully, the assistant enters a special post-creation state that guides you through essential setup steps.

Progress Indicator

The interface shows a progress indicator: 


Target -> Describe -> Apply -> Wheels -> Colliders

                              ^^^^^^^^

                           (You are here)

This helps track which steps are complete and what remains.

What's Already Done

At this point, your vehicle has:

  • RCCP_CarController added and configured
  • Engine, gearbox, differential components set up
  • Basic WheelCollider components created
  • Physics properties configured
  • Input system ready

What Remains

You need to complete:

  1. Wheel Assignment - Confirm which wheel meshes match which positions
  2. Body Colliders - Add collision geometry for physics

Wheel Assignment Wizard

The wheel assignment wizard ensures your wheel meshes are correctly mapped to RCCP's wheel positions.

Why It's Needed

RCCP needs to know:

  • Which wheel is front-left vs front-right vs rear-left vs rear-right
  • Where to position WheelColliders
  • Which wheels to apply steering to
  • Which wheels receive power (based on drivetrain)

Auto-Detection

The assistant attempts automatic detection based on:

  • Wheel positions relative to vehicle center
  • Forward direction (Z+) for front/rear
  • Left/Right position (X axis) for side

Detection Logic:

  • Wheels ahead of center = Front
  • Wheels behind center = Rear
  • Wheels on negative X = Left
  • Wheels on positive X = Right

Reviewing Auto-Detection

The wizard shows four assignment slots: 


Front Left (FL):  [WheelFL_Mesh]     [Change]

Front Right (FR): [WheelFR_Mesh]     [Change]

Rear Left (RL):   [WheelRL_Mesh]     [Change]

Rear Right (RR):  [WheelRR_Mesh]     [Change]

Each slot shows:

  • Position label
  • Currently assigned wheel name
  • Change button to reassign

Confirming Assignments

If auto-detection is correct:

  1. Verify each wheel name looks right
  2. Click Confirm Assignments
  3. The wizard proceeds to the next step

Manual Reassignment

If a wheel is incorrectly assigned:

  1. Click Change next to the wrong assignment
  2. A dropdown or picker appears with available wheel transforms
  3. Select the correct wheel
  4. Repeat for any other incorrect assignments
  5. Click Confirm Assignments

Common Assignment Issues

All wheels on one side

Your model may have non-standard orientation. Check the forward direction.

Wheels swapped front/rear

The model's Z-axis may be pointing backward. You may need to rotate the model or parent.

Wheels swapped left/right

Check if your model uses a different coordinate convention.

Missing wheels

Some wheel meshes weren't detected. Ensure they're separate GameObjects with "wheel" in the name.

Six-Wheel Vehicles

For vehicles with more than four wheels (trucks, buses):

  • The wizard focuses on the primary four wheels
  • Additional wheels can be configured manually in RCCP components
  • Front-most and rear-most pairs are typically auto-detected

Body Colliders Wizard

After wheel assignment, the body colliders wizard helps add physics collision geometry.

Why Colliders Matter

Without colliders:

  • The vehicle passes through objects
  • Damage detection doesn't work
  • Physics interactions fail
  • Flipping/rolling behavior is incorrect

Collider Options

The wizard offers several collider types:

Box Colliders

  • Simplest option
  • Fast performance
  • Less accurate for curved shapes
  • Good for: Boxy vehicles, prototyping

Convex Mesh Colliders

  • Follows mesh shape
  • Convex hull approximation
  • Good balance of accuracy and performance
  • Good for: Most vehicles

Multiple Primitive Colliders

  • Several boxes/capsules combined
  • Manual positioning required
  • Accurate and performant
  • Good for: Complex shapes

Quick Setup

For rapid setup:

  1. Click Add Basic Colliders
  2. The wizard generates box colliders based on mesh bounds
  3. Adjust positions/sizes in the Inspector if needed
  4. Click Done

Custom Setup

For more control:

  1. Click Advanced Setup
  2. Choose collider type (Box, Convex, etc.)
  3. Configure generation settings
  4. Preview the result
  5. Adjust individual colliders
  6. Click Finish Setup

Post-Collider Adjustments

After the wizard completes, you can still adjust colliders:

  1. Select the vehicle in Hierarchy
  2. Expand to find collider GameObjects
  3. Use standard Unity tools to resize/reposition
  4. Consider physics layers for different collision behaviors

Transitioning to Customization

After completing post-creation setup, you have several paths forward:

Continue Customizing

The assistant offers to continue with the Vehicle Customization panel:

  1. A prompt asks "Continue to customization?"
  2. Click Yes to switch panels
  3. Your newly created vehicle remains selected
  4. Write customization requests to refine the vehicle

Test Your Vehicle

Before extensive customization:

  1. Add a ground plane if not present
  2. Enter Play mode
  3. Test basic driving behavior
  4. Note what needs adjustment
  5. Exit Play mode and customize accordingly

Use Specialized Panels

Jump directly to specific aspects:

  • Behaviors - Adjust handling characteristics
  • Wheels - Fine-tune grip and suspension
  • Audio - Configure engine sounds
  • Lights - Set up vehicle lighting
  • Damage - Configure damage system

Run Diagnostics

Validate your new vehicle:

  1. Switch to the Diagnostics panel
  2. Click Run Diagnostics
  3. Review any warnings or issues
  4. Address problems before extensive customization

Common Post-Creation Tasks

Adjusting Center of Mass

If the vehicle tips over easily:

  1. Select the vehicle
  2. In RCCP_CarController, adjust Center of Mass offset
  3. Lower Y value = more stability
  4. Test and iterate

Adding More Colliders

For complex collision needs:

  1. Create empty child GameObjects
  2. Add collider components
  3. Position and size appropriately
  4. Consider trigger colliders for special zones

Setting Up Physics Layers

For proper collision filtering:

  1. Create layers for vehicle body and wheels
  2. Assign layers to colliders
  3. Configure Physics collision matrix
  4. Prevent unwanted collisions

Tips

Test Early and Often

Don't wait until everything is "perfect" to test. Early testing reveals issues.

Save a Backup

After successful creation, save the scene. You can always revert if later changes go wrong.

Use Prefabs Carefully

Don't make the vehicle a prefab until post-creation is complete. RCCP setup can behave differently with prefabs.

Document Your Configuration

Note what description you used. This helps reproduce or reference the configuration later.

Troubleshooting

Vehicle Falls Through Ground

Missing or incorrectly positioned wheel colliders. Re-run wheel assignment or check collider setup.

Vehicle Flips Immediately

Center of mass too high or colliders intersecting ground. Adjust COM or collider positions.

Wheels Don't Touch Ground

WheelCollider radius or suspension settings incorrect. Check wheel configuration in RCCP_Axles.

Vehicle Doesn't Move

Input not configured or drivetrain not connected. Run diagnostics to identify issues.

Steering Doesn't Work

Front wheels not marked for steering. Check RCCP_Axle steering settings.
