The Vehicle Customization panel allows you to modify existing RCCP vehicles using natural language descriptions. Unlike Vehicle Creation which starts from raw models, customization works with vehicles that already have RCCP components.

Prerequisites

What You Need

An Existing RCCP Vehicle

The selected GameObject must have an RCCP_CarController component. This can be:

  • A vehicle created through the AI Assistant
  • A vehicle from RCCP demo scenes
  • A manually configured RCCP vehicle
  • A vehicle prefab instantiated in the scene

Scene Object

The vehicle must be in the scene hierarchy, not just a prefab in the Project window.

Active State

The vehicle's GameObject should be active (not disabled).

Recognizing RCCP Vehicles

When you select a valid RCCP vehicle, the assistant shows:

  • Vehicle name with RCCP indicator
  • Currently attached components
  • Configuration summary

If you select something that's not an RCCP vehicle, the assistant prompts you to select a valid target.

Workflow

Step 1: Select Target Vehicle

  1. In the Hierarchy, select your RCCP vehicle
  2. The AI Assistant recognizes it and shows vehicle info
  3. Current configuration is captured for comparison

The assistant displays:

  • Vehicle name
  • Component list
  • Key current values (power, drivetrain, etc.)

Step 2: Describe Your Changes

Write a natural language description of what you want to change.

Tip: If you're unsure what to change, switch to Ask Mode first. Click the "Ask" button at the top of the prompt area to ask questions like "What might cause understeer?" or "What power level is appropriate for this vehicle?" Then switch back to Configure mode to make changes. See the Ask Mode documentation for details.

Good customization prompts include:

  • What aspect to change (power, handling, stability, etc.)
  • Direction of change (increase, decrease, enable, disable)
  • Magnitude if relevant (small adjustment vs major change)
  • Context or goal (why you want this change)

Examples:

Increasing Power: 


Increase engine power to 350hp and raise the rev limit to 7500 RPM.

Adjust torque curve for more low-end pull.

Improving Handling: 


Make the steering more responsive at low speeds but more stable

at highway speeds. Reduce body roll in corners.

Adjusting Stability: 


Enable ABS with moderate intervention. Add light ESP that allows

some slip before correction. Keep TCS off for manual control.

Changing Drivetrain: 


Convert to all-wheel drive with 40/60 front/rear torque split.

Adjust the differential for better corner exit traction.

Step 3: Review Changes

The AI generates a response showing proposed modifications.

Before/After Comparison

For each changed property, you see:

  • Property name
  • Current value (before)
  • Proposed value (after)

Component Organization

Changes are grouped by component:

  • Engine changes
  • Gearbox changes
  • Stability changes
  • etc.

Selective Toggles

Each component group has a toggle to include/exclude it from the apply operation.

Step 4: Apply Changes

Choose how to apply:

Apply All

Applies every change in the response.

Apply Selected

Only applies changes from enabled component groups.

Discard

Clears the response without making changes.

After applying:

  • Changes are made to the vehicle
  • Undo is available (Ctrl+Z)
  • Status message confirms success

What Can Be Customized

Performance

Engine

  • Maximum power (horsepower/kW)
  • Maximum torque
  • RPM range (idle to redline)
  • Torque curves
  • Engine inertia

Transmission

  • Gearbox type (auto/manual)
  • Number of gears
  • Gear ratios
  • Shift timing
  • Clutch behavior

Drivetrain

  • Drive type (FWD/RWD/AWD)
  • Torque distribution
  • Differential settings
  • Final drive ratio

Handling

Steering

  • Steering angle limits
  • Speed-sensitive steering curves
  • Counter-steer assistance
  • Steering response

Suspension

  • Spring stiffness
  • Damper rates
  • Suspension travel
  • Anti-roll bar effectiveness

Weight Distribution

  • Center of mass position
  • Vehicle mass
  • Inertia settings

Stability Systems

ABS (Anti-lock Braking)

  • Enable/disable
  • Intervention level
  • Sensitivity

ESP (Electronic Stability Program)

  • Enable/disable
  • Correction strength
  • Activation threshold

TCS (Traction Control)

  • Enable/disable
  • Slip limit
  • Power reduction rate

Other Aspects

Tires

  • Friction coefficients
  • Grip multipliers
  • Slip characteristics

Brakes

  • Brake force
  • Brake bias (front/rear)

For specialized aspects (audio, lights, damage), use the dedicated panels for more focused control.

Customization vs. Specialized Panels

When to Use Vehicle Customization

Use this panel for:

  • Broad changes across multiple components
  • Performance tuning (power, handling)
  • General driving feel adjustments
  • When you're not sure which specific component to modify

When to Use Specialized Panels

Use dedicated panels for:

  • Behaviors - Detailed stability and driving style presets
  • Wheels - Precise tire and suspension tuning
  • Audio - Engine sound configuration
  • Lights - Vehicle lighting setup
  • Damage - Deformation and detachable parts

Specialized panels have:

  • Focused context for that domain
  • More relevant quick prompts
  • Optimized AI prompts for specific changes

State Snapshot System

The customization panel captures your vehicle's state before generating changes.

Before State

When you click Generate, the current values of all customizable properties are recorded.

After State

The AI response proposes new values.

Comparison View

You see both states side-by-side, making it easy to understand what will change.

Undo Safety

If applied changes don't work as expected, Unity's Undo system restores the before state.

Tips for Effective Customization

Be Specific About Direction

Say "increase power by 50hp" rather than just "more power."

Describe the Problem

Instead of guessing a solution, describe what's wrong: "The car understeers too much in tight corners."

Reference Behavior

Describe how you want it to feel: "Like a go-kart with quick turn-in."

Iterate Gradually

Make smaller changes and test. Large changes can have unexpected interactions.

Use Test Drives

After each change, test in Play mode. Real driving feel is the ultimate validator.

Combine with Manual Tuning

Use AI for initial changes, then fine-tune manually in inspectors for precision.

Common Customization Scenarios

Making a Vehicle Faster 


Increase engine power to 400hp, add a 6th gear for higher top speed,

and reduce vehicle mass by 100kg for better acceleration.

Improving Stability 


Lower the center of mass, stiffen the suspension, and enable ESP

with moderate intervention. The car should resist spinning out.

Creating a Drift Setup 


Disable TCS and ESP. Set front grip to 1.1 and rear grip to 0.9

for controllable slides. Increase steering angle and enable

counter-steer assistance.

Comfort Tuning 


Soften the suspension significantly, reduce steering response for

less twitchiness, and smooth out the throttle response curve.

Track Preparation 


Stiffen suspension, lower ride height, increase downforce effect,

enable aggressive ABS, and sharpen throttle response.

Troubleshooting

Changes Don't Feel Different

Some changes have subtle effects. Compare before/after values and test in appropriate conditions.

Vehicle Behaves Erratically

Aggressive changes can interact unpredictably. Undo and try smaller adjustments.

AI Misunderstands Intent

Rephrase your prompt more specifically. Mention exact components or values if needed.

Can't Undo Changes

Unity's undo stack may fill up. Save your scene before major customization sessions.

Some Properties Not Changed

The AI focuses on properties relevant to your description. For complete overhauls, make multiple requests or use specialized panels.
