The Diagnostics panel performs local validation of RCCP vehicles without using AI. It checks for common issues, missing components, and configuration problems to help ensure your vehicles are properly set up.

What This Panel Does

Local Validation

Unlike other panels that use AI to generate configurations, Diagnostics runs entirely locally:

  • No API calls required
  • No usage of your request quota
  • Instant results
  • Works offline

What Gets Checked

The diagnostic system examines:

Vehicle Structure

  • RCCP_CarController presence and state
  • Required component hierarchy
  • GameObject structure

Component Presence

  • Required components (Engine, Gearbox, etc.)
  • Optional components and their state
  • Component references and connections

Configuration Values

  • Invalid or extreme values
  • Missing required values
  • Conflicting settings

Physical Setup

  • Rigidbody configuration
  • WheelCollider setup
  • Collider presence and types

References

  • Missing or broken references
  • Orphaned components
  • Circular dependencies

Running Diagnostics

How to Use

  1. Select an RCCP vehicle in the scene
  2. Switch to the Diagnostics panel
  3. Click Run Diagnostics
  4. Review the results

There's no prompt input - diagnostics are automatic based on the vehicle's current state.

Results Display

Results are categorized by severity:

Info (Blue)

Informational messages that don't indicate problems:

  • Configuration details
  • Statistics
  • Notes

Warning (Yellow)

Potential issues that may or may not cause problems:

  • Unusual but valid configurations
  • Missing optional components
  • Performance concerns

Error (Red)

Actual problems that need attention:

  • Missing required components
  • Invalid configurations
  • Broken references

Filtering Results

Toggle filters to show/hide severity levels:

  • Show/Hide Info messages
  • Show/Hide Warnings
  • Show/Hide Errors

This helps focus on what matters to you.

Auto-Fix Feature

Some common issues can be automatically fixed without manual intervention.

How Auto-Fix Works

When diagnostics identify fixable issues:

  1. An Auto-Fix button appears in the results area
  2. Click the button to attempt automatic repairs
  3. The system fixes what it can and reports results
  4. Diagnostics run again to verify fixes

What Can Be Auto-Fixed

The auto-fix system can resolve:

Missing References

  • Reconnect broken component references
  • Link orphaned wheel colliders
  • Restore axle-to-wheel connections

Invalid Values

  • Reset zero or negative values to valid defaults
  • Correct out-of-range settings
  • Fix inverted min/max values

Component Setup

  • Add missing required child components
  • Initialize unset required fields
  • Correct component hierarchy issues

What Cannot Be Auto-Fixed

Some issues require manual attention:

Structural Problems

  • Missing wheel meshes in the model
  • Incorrect model scale or orientation
  • Merged wheels that need separation

Design Decisions

  • Center of mass positioning
  • Collider shapes and sizes
  • Physics layer assignments

Asset References

  • Missing audio clips
  • Missing materials
  • Missing prefab references

Per-Issue Fix Buttons

Individual issues that can be fixed show a small fix button: 


[!] WheelCollider radius is zero        [Fix]

    Suggestion: Set radius to match wheel mesh

Click the individual fix button to resolve just that issue.

Auto-Fix Safety

Auto-fix operations are safe:

  • All changes support Unity Undo (Ctrl+Z)
  • Original values are logged to Console
  • Fixes use RCCP default values when appropriate
  • Prefab instances are handled correctly

Recommended Workflow

  1. Run diagnostics first
  2. Review the error list
  3. Click Auto-Fix to resolve what's possible
  4. Manually address remaining issues
  5. Run diagnostics again to confirm

Common Diagnostics

Structure Checks

RCCP_CarController Status

  • Present and enabled
  • Properly initialized
  • References valid

Hierarchy Check

  • Standard RCCP hierarchy maintained
  • No unexpected parents or children
  • Proper nesting of components

Component Checks

Required Components:

  • RCCP_Engine - Power source
  • RCCP_Gearbox - Transmission
  • RCCP_Differential - Power distribution
  • RCCP_Axles - Wheel connections
  • RCCP_Input - Control input

Optional Components:

  • RCCP_Stability - ABS/ESP/TCS
  • RCCP_Audio - Engine sound
  • RCCP_Lights - Vehicle lighting
  • RCCP_Damage - Damage system

Configuration Checks

Engine

  • Power values within reasonable range
  • RPM range valid (idle < max)
  • Torque curve defined

Gearbox

  • Gear count matches gear ratios
  • Ratios are valid numbers
  • Final drive defined

Wheels

  • WheelColliders present for each wheel
  • Suspension values positive
  • Friction curves defined

Physical Checks

Rigidbody

  • Present on main vehicle body
  • Mass is reasonable
  • Drag values appropriate
  • Center of mass configured

Colliders

  • At least one collider on body
  • No inverted or zero-size colliders
  • Appropriate collision layers

WheelColliders

  • Correct number (typically 4)
  • Radius appropriate for wheel mesh
  • Suspension travel defined

Reference Checks

Component References

  • All required references assigned
  • No null references where required
  • No circular references

Asset References

  • Audio clips assigned (if audio enabled)
  • Materials/textures present
  • Prefab references valid

Interpreting Results

Info Messages

"Vehicle has X components configured"

Summary of what's set up. Not a problem.

"Using default values for Y"

Component is using RCCP defaults. May want to customize.

"Optional component Z not present"

An optional feature isn't configured. Add if desired.

Warning Messages

"Engine power unusually high/low"

Power value is outside typical range. May be intentional for special vehicles.

"Suspension travel very short"

May bottom out easily. Could be intentional for low-rider style.

"Center of mass not configured"

Using automatic calculation. May want to fine-tune.

"No audio configured"

Vehicle will be silent. Add audio if desired.

Error Messages

"Missing RCCP_Engine component"

Required component not found. Vehicle won't function.

"WheelCollider radius is zero"

Invalid configuration. Wheel won't work correctly.

"Null reference in axle configuration"

A required reference is missing. Needs to be assigned.

"Rigidbody mass is zero"

Physics won't work correctly. Set a valid mass.

When to Run Diagnostics

After Vehicle Creation

Run diagnostics immediately after creating a vehicle to catch any setup issues:

  1. Complete vehicle creation
  2. Finish post-creation setup
  3. Run diagnostics
  4. Address any errors before testing

After Manual Changes

If you've made manual changes in the Inspector:

  1. Make your changes
  2. Run diagnostics
  3. Verify nothing was broken

When Something Doesn't Work

If your vehicle behaves unexpectedly:

  1. Run diagnostics first
  2. Check for errors
  3. Address problems before investigating further

Before Shipping

As a final check before releasing:

  1. Run diagnostics on all vehicles
  2. Ensure no errors
  3. Review warnings for acceptability

Diagnostic Categories

Critical Issues

Must be fixed for the vehicle to function:

  • Missing required components
  • Null references
  • Zero or negative physics values
  • Broken component connections

Important Issues

Should usually be fixed:

  • Unusual value ranges
  • Missing common components
  • Configuration inconsistencies

Minor Issues

May or may not need attention:

  • Missing optional features
  • Using default values
  • Non-standard configurations

Tips for This Panel

Use Frequently

Diagnostics are free (no API cost). Run them often during development.

Fix Errors First

Always address red errors before worrying about warnings.

Understand Warnings

Some warnings are intentional for special vehicles. Know why they appear.

Use with Other Panels

After using other panels to modify a vehicle, run diagnostics to verify the result.

Keep Vehicles Clean

Regular diagnostics help maintain vehicle quality throughout development.

Pre-Flight Check

Make diagnostics part of your workflow before each play test.

Troubleshooting Common Issues

"Missing RCCP_CarController"

The selected object isn't an RCCP vehicle. Select a proper vehicle or create one first.

"WheelColliders not found"

Wheels weren't set up properly. Re-run post-creation wheel wizard or manually add WheelColliders.

"Mass not set"

The Rigidbody needs a mass value. Set it manually or regenerate vehicle configuration.

"Invalid gear ratios"

Gearbox configuration has problems. Check for zero or negative ratios.

"Suspension travel invalid"

WheelCollider suspension values are wrong. Ensure positive travel distances.

Many warnings after creation

Some warnings are expected for newly created vehicles before full customization. Focus on errors first.

Auto-Fix did nothing

If Auto-Fix reports "No auto-fixable issues found":

  • The detected issues may require manual intervention
  • Check if issues are warnings vs errors (some warnings aren't fixable)
  • The issues may be design decisions rather than errors

Auto-Fix made things worse

Use Undo (Ctrl+Z) immediately to revert. Auto-fix uses safe defaults, but your specific configuration may need different values.
