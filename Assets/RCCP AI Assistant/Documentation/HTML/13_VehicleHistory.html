The Vehicle History feature tracks all AI-generated modifications made to each vehicle. This per-vehicle history allows you to review past changes and restore previous configurations if needed.

What Is Vehicle History

Per-Vehicle Tracking

Each RCCP vehicle maintains its own modification history:

  • History is specific to that vehicle instance
  • Stored as a component on the vehicle GameObject
  • Persists across Unity sessions (saved with scene)

What Gets Recorded

Each history entry captures:

  • Timestamp of the modification
  • Panel used (Creation, Customization, Behaviors, etc.)
  • User prompt that generated the change
  • AI response (configuration JSON)
  • Before state snapshot
  • After state snapshot
  • Whether changes were applied

The RCCP_AIHistory Component

History is stored in a component automatically added to vehicles:

  • Added when first modification is tracked
  • Visible in Inspector (read-only)
  • Serialized with the vehicle

Viewing Vehicle History

Accessing History

  1. Select an RCCP vehicle in the scene
  2. In the AI Assistant, history is accessible via the sidebar
  3. Click the vehicle history icon/button
  4. The history panel opens showing all past modifications

History Panel Layout

The panel shows:

  • Vehicle name at the top
  • List of history entries (newest first)
  • Each entry shows timestamp, panel, and summary
  • Click an entry to expand details

Entry Details

Expanding an entry shows:

  • Full user prompt
  • Complete AI response
  • Before/After state comparison
  • Actions (restore, view diff)

History Entry Information

Timestamp

When the modification occurred:

  • Date and time
  • Relative time ("2 hours ago")

Panel Type

Which panel generated the change:

  • Vehicle Creation
  • Vehicle Customization
  • Behaviors
  • Wheels
  • Audio
  • Lights
  • Damage

User Prompt

The exact prompt that generated this configuration:

  • Full text preserved
  • Useful for understanding intent
  • Can be copied and reused

AI Response

The complete JSON response from the AI:

  • All configuration values
  • Collapsible for readability
  • Can be copied to clipboard

Before State

A snapshot of the vehicle's configuration before the change:

  • Component values at that moment
  • Enables comparison with after state
  • Essential for restore functionality

After State

The vehicle's configuration after applying the change:

  • Shows what was actually applied
  • May differ slightly from AI response if selective apply was used

Restoring Previous Configurations

How Restore Works

Restoring reverts the vehicle to a previous state:

  1. Before state from history entry is loaded
  2. Current configuration is overwritten
  3. Vehicle reflects the restored state

Restore Process

  1. Open vehicle history
  2. Find the entry you want to restore to
  3. Click Restore to This State
  4. Confirm the action
  5. Vehicle configuration is reverted

What Gets Restored

Restoration affects:

  • All RCCP component values
  • Properties that were captured in the before state
  • Does not affect non-RCCP components

Partial Restore

You can selectively restore components:

  • Expand the before state
  • Toggle which components to restore
  • Apply partial restoration

Viewing Differences

Diff Visualization

Compare two states side-by-side:

  • Before state on one side
  • After state on other side
  • Changed values highlighted

Understanding the Diff

The diff shows:

  • Added: Values that didn't exist before (green)
  • Removed: Values that were removed (red)
  • Changed: Values that were modified (yellow)
  • Unchanged: Values that stayed the same (gray)

Using Diffs

Diffs help you:

  • Understand what a modification actually changed
  • Identify unexpected changes
  • Learn how different prompts affect configuration

Managing History

History Storage

History is stored:

  • As serialized data in RCCP_AIHistory component
  • In the scene file (saved with scene)
  • Per-vehicle, not globally

History Size

By default, history keeps:

  • Last 50 entries per vehicle
  • Older entries are removed when limit is reached
  • Adjustable in settings

Clearing History

To clear a vehicle's history:

  1. Open vehicle history
  2. Click Clear History
  3. Confirm the action
  4. All entries are removed

This is irreversible - cleared history cannot be recovered.

Selective Deletion

To remove specific entries:

  1. Open vehicle history
  2. Select entries to delete
  3. Click Delete Selected
  4. Confirm the action

Use Cases

Undoing Recent Changes

If a recent change made things worse:

  1. Open history
  2. Find the last good entry
  3. Restore to that state
  4. Continue from there

Comparing Approaches

If you tried multiple prompts:

  1. Review history entries
  2. Compare their before/after states
  3. Choose the best result
  4. Restore to preferred state

Learning from Past Prompts

To understand what works:

  1. Browse successful history entries
  2. Study the prompts that produced good results
  3. Adapt successful prompts for future use

Reverting Experiments

After experimental changes:

  1. Find pre-experiment entry
  2. Restore to before the experiment
  3. Vehicle is back to stable state

Documenting Configuration Process

History serves as documentation:

  • Shows how the vehicle evolved
  • Records the prompts used
  • Provides before/after context

Best Practices

Before Major Changes

Before significant modifications:

  • Note the current state's position in history
  • Make changes
  • If unsatisfied, restore to the noted state

Regular Saves

While history provides restore capability:

  • Still save your scene regularly
  • History relies on scene being saved
  • Unsaved history is lost if Unity crashes

Meaningful Prompts

Write descriptive prompts:

  • History shows the prompt text
  • Clear prompts make history more useful
  • Future you will thank present you

Review Before Restore

Before restoring:

  • Check what state you're restoring to
  • Verify it's the right entry
  • Understand what will change

Clean Up Periodically

If history gets cluttered:

  • Remove failed experiments
  • Keep significant milestones
  • Maintain a clean history for easier navigation

Troubleshooting

History Not Appearing

If a vehicle has no history:

  • It may not have been modified through the AI Assistant
  • Manual Inspector changes aren't tracked
  • History component may not have been added

Missing Before State

If before state is incomplete:

  • State capture may have failed
  • Some component values may not serialize
  • Check if component was present at modification time

Restore Not Working

If restore seems to have no effect:

  • Check if values are actually different
  • Verify the component still exists
  • Ensure scene can be modified (not read-only)

History Lost

If history disappears:

  • Check if RCCP_AIHistory component was removed
  • Verify scene was saved after modifications
  • Scene may have been reverted or reloaded

Integration with Other Features

Prompt History vs Vehicle History

Vehicle History:

  • Per-vehicle
  • Includes before/after states
  • Enables restore
  • Stored in scene

Prompt History:

  • Global (all vehicles)
  • Just prompts and responses
  • Enables re-apply to other vehicles
  • Stored in EditorPrefs

Using Both Together

  1. Use Vehicle History to track a specific vehicle's evolution
  2. Use Prompt History to find successful prompts
  3. Apply prompts from history to different vehicles
  4. Each application creates vehicle history entries
