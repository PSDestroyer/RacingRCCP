This guide walks you through creating your first AI-configured vehicle in RCCP AI Assistant. By the end, you'll have a fully functional vehicle and understand the basic workflow.

Opening the Assistant

  1. In Unity, go to Tools > BoneCracker Games > RCCP AI Assistant > Open Assistant
  2. The AI Assistant window will open

If this is your first time, you'll see the Welcome panel. Take a moment to read through it, then click "Get Started" to proceed.

Choosing Your API Mode

Before using the AI features, you need to configure how the assistant accesses the Claude AI. You have two options:

Option A: Server Proxy (Recommended for Getting Started)

  • No API key needed
  • Free tier included (400 setup requests at 20/hour, then 20/day at 10/hour)
  • Automatic device registration
  • Just click "Use Server Proxy" in Settings

Option B: Own API Key

  • Unlimited requests
  • Pay per usage directly to Anthropic
  • Requires Claude API key from console.anthropic.com
  • Enter your key in Settings > API Configuration

For first-time users, the Server Proxy is the easiest way to start. You can always switch to your own API key later.

The Three-Step Workflow

All vehicle creation and customization follows the same three-step pattern: 


1. Select Target  ->  2. Describe  ->  3. Review & Apply

This keeps you in control while the AI handles the technical complexity.

Creating Your First Vehicle

Prerequisites

Before creating a vehicle, you need:

  • A 3D car model in your scene (not a prefab in Project)
  • The model should have separate wheel meshes
  • Proper real-world scale (a typical sedan is 4-5 meters long)
  • Z-axis pointing forward, Y-axis pointing up

Step 1: Select Your Model

  1. In the Hierarchy window, select your 3D car model's root GameObject
  2. In the AI Assistant window, select the Vehicle Creation panel from the sidebar
  3. The assistant will analyze your selection and show:
  • Model dimensions
  • Detected wheel count
  • Any eligibility warnings

If you see green checkmarks for Scale, Orientation, and Wheels, your model is ready.

Step 2: Describe Your Vehicle

In the text area, describe what kind of vehicle you want to create. Be specific about:

  • Vehicle type (sports car, sedan, truck, etc.)
  • Performance characteristics (fast, stable, powerful, etc.)
  • Handling feel (responsive, comfortable, drift-ready, etc.)

Example prompts:

For a sports car: 


High-performance sports car with 400 horsepower, rear-wheel drive,

responsive steering, and firm suspension for track use.

For a family sedan: 


Comfortable family sedan with smooth acceleration, stable handling,

all-wheel drive, and forgiving suspension for daily driving.

For a drift car: 


Rear-wheel drive drift car with high power, loose rear end,

quick steering response, and reduced stability control.

Click Generate (or press Enter if enabled in settings).

Step 3: Review and Apply

The AI will generate a configuration. Before applying, you'll see:

  1. Proposed Changes - A structured view of all components and their values
  2. Selective Apply Options - Toggle which component groups to apply
  3. Action Buttons - Apply All, Apply Selected, or Discard

Review the configuration:

  • Check that engine power matches your expectations
  • Verify drivetrain type (FWD/RWD/AWD)
  • Look at stability settings

When satisfied, click Apply to create the vehicle.

Post-Creation Setup

After clicking Apply, the assistant adds RCCP components to your model and enters post-creation mode:

Wheel Assignment Wizard

The assistant needs to know which wheel meshes correspond to which wheel positions:

  1. You'll see a wheel assignment panel with four positions:
  • Front Left (FL)
  • Front Right (FR)
  • Rear Left (RL)
  • Rear Right (RR)
  1. The AI attempts auto-detection based on wheel positions. If correct, click Confirm Assignment.
  1. If auto-detection is wrong, click each button and select the correct wheel transform from your model's hierarchy.

Body Colliders Wizard

After wheels are assigned, you can add physics colliders:

  1. Click Add Body Colliders to open the wizard
  2. Choose collider types (box, convex mesh, etc.)
  3. Adjust as needed for your model's shape
  4. Click Apply Colliders

Proper colliders are essential for realistic physics and collision detection.

Testing Your Vehicle

  1. Add a ground plane to your scene if not present
  2. Enter Play mode
  3. Your vehicle should be drivable with RCCP's default input settings

If the vehicle doesn't behave as expected, you can return to the AI Assistant and use the Vehicle Customization panel to make adjustments.

Understanding the Workflow: A Summary

  1. Select - Choose a raw 3D model (for creation) or existing RCCP vehicle (for customization)
  2. Describe - Write natural language describing what you want
  3. Review - See proposed changes with before/after comparisons
  4. Apply - Commit changes to your vehicle
  5. Post-Setup - Assign wheels and colliders (creation only)

Quick Tips

Writing Effective Prompts

Be specific about performance:

  • Instead of "fast car" say "sports car with 350hp and 6-speed manual"
  • Instead of "stable" say "highway cruiser with soft suspension and strong stability control"

Mention handling characteristics:

  • "responsive steering at low speeds"
  • "predictable understeer at the limit"
  • "loose rear end for drifting"

Reference real vehicles for context:

  • "handling similar to a BMW M3"
  • "comfort-focused like a luxury sedan"

Using Quick Prompts

Each panel has quick prompt buttons with common requests. These are great starting points - click one and modify it to fit your needs.

Next Steps

Now that you've created your first vehicle:

  1. Experiment with Customization - Select your vehicle and try the customization panels
  2. Read API Configuration - Learn about the different API modes in detail
  3. Explore Panel Documentation - Each panel has specific features documented
  4. Check Prompt History - Your prompts are saved for future reference

The AI Assistant is designed for iteration. Don't expect perfection on the first try - use the conversational workflow to refine your vehicles until they feel right.
