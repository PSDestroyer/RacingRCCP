When using the server proxy (not your own API key), RCCP AI Assistant implements a two-phase tier system to provide fair access while protecting shared resources. This document explains how the system works and how to manage your usage effectively.

Overview

The usage limit system has two phases:

  1. Setup Phase (Tier 1): Generous initial pool for evaluation and setup
  2. Free Phase (Tier 2): Sustainable daily limits for ongoing use

If you use your own API key, these limits don't apply - you have unlimited requests.

Two-Phase Tier System

Phase 1: Setup Phase

When you first register a device, you receive a setup pool of requests with no daily limits.

MetricValue
Total Requests400
Daily LimitNone
Hourly Burst20 per hour
ExpirationNever (until exhausted)

Purpose: The setup pool lets you thoroughly evaluate the tool and set up initial vehicles without worrying about daily limits. You can use all 400 requests in a single day if needed.

Best Practices:

  • Use the setup phase for intensive initial configuration
  • Experiment with different prompts to learn what works
  • Set up your main vehicles during this phase
  • Take advantage of no daily limits for iteration

Phase 2: Free Phase

After exhausting your setup pool, you transition to the free phase with daily limits.

MetricValue
Daily Requests20
Hourly Burst10 per hour
Daily ResetMidnight (local time)
Hourly ResetEvery 60 minutes

Purpose: The free phase provides sustainable ongoing access for light usage and maintenance tasks.

Best Practices:

  • Plan your daily requests to maximize value
  • Use diagnostics (no AI cost) for validation
  • Consider using your own API key for intensive work

Transition Notification

When you exhaust your setup pool, a dialog appears explaining:

  • Your setup phase is complete
  • You now have 20 free requests per day
  • Consider using your own API key for unlimited access

This notification appears once to ensure you understand the change.

Reset Schedule

Daily Limit Reset

The daily limit resets at midnight local time on your machine. After reset:

  • Daily used counter returns to 0
  • You have 20 fresh requests for the new day

Note: The server uses your device's reported timezone for reset timing.

Hourly Burst Reset

The hourly burst limit resets 60 minutes after your first request of each hour. This prevents rapid consecutive requests while still allowing sustained usage throughout the day.

Example timeline:

  • 10:00 AM - First request (19 remaining during setup, 9 during free phase)
  • 10:15 AM - Second request (18 or 8 remaining)
  • 10:30 AM - Third request (17 or 7 remaining)
  • 11:00 AM - Hourly reset (20 or 10 available again)

Server-Authoritative

All limits are enforced server-side. The local UI displays cached values for convenience, but the server has the final say on whether a request is allowed. This prevents tampering with local counters.

Settings Panel Breakdown

Open Settings for detailed usage information:

  • Current Tier: Setup Phase, Daily Free, or Own API Key
  • Setup Pool: Remaining/Total (if in setup phase)
  • Daily Requests: Used/Limit with time until reset
  • Hourly Burst: Used/Limit with time until reset
  • Total Requests: Lifetime count across all tiers
  • Last Sync: When usage was last updated from server

Progress Bars

Visual progress bars show usage at a glance:

  • Green: More than 50% remaining
  • Yellow: 20-50% remaining
  • Red: Less than 20% remaining

What Happens When Limits Are Reached

Daily Limit Reached

When you hit the daily limit:

  1. A dialog appears explaining the situation
  2. The Generate button becomes disabled
  3. The context bar shows "Daily limit reached"

Options:

  • Wait until midnight for reset
  • Switch to your own API key for immediate access
  • Use Diagnostics panel (no AI required)

Hourly Burst Reached

When you hit the hourly burst limit:

  1. The UI shows time until hourly reset
  2. You can still make requests up to your daily limit after the hour passes

Note: Hourly limits prevent rapid automated usage but shouldn't impact normal human interaction patterns.

Setup Pool Exhausted

When you use your last setup request:

  1. Transition dialog appears (once)
  2. System switches to Free Phase
  3. Daily limits now apply

Tips for Managing Usage

Maximize Each Request

Write Complete Prompts

Instead of iterating with vague requests, write detailed prompts that describe exactly what you want. One well-crafted prompt often beats three vague ones.

Use Appropriate Panels

Select the panel that matches your task. The Wheels panel has context about wheel-specific options that might require multiple generic requests to achieve.

Leverage Quick Prompts

Quick prompt buttons provide tested, effective starting points. Modify them rather than starting from scratch.

Use Free Features

Diagnostics Panel

The Diagnostics panel runs local validation without any AI calls. Use it freely to check vehicle setup.

Review Before Applying

Take time to review AI responses before applying. Catching issues before applying saves re-generation requests.

Prompt History

Browse your prompt history to re-use successful prompts instead of writing new ones.

Plan Intensive Work

If you need to do intensive vehicle setup:

  1. Do it during your Setup Phase when possible
  2. Consider switching to your own API key temporarily
  3. Batch similar vehicles together using batch operations

Upgrading to Unlimited

If the free tier doesn't meet your needs, using your own API key provides unlimited access:

  1. Create account at console.anthropic.com
  2. Add a payment method
  3. Generate an API key
  4. Enter the key in Settings > API Configuration
  5. Toggle "Use Own API Key" to enabled

With your own key:

  • No request limits
  • No daily resets to wait for
  • Pay only for what you use
  • Choose any Claude model

FAQ

Can I get more setup requests?

The setup pool is per-device and doesn't reset. If you need more requests, consider using your own API key.

Do unused daily requests carry over?

No, daily allowances reset to 20 each midnight regardless of how many you used the previous day.

What counts as a request?

Each time you click Generate and receive a response, that's one request. Failed requests that return errors may not count, depending on where the failure occurred.

Do Diagnostics use requests?

No, the Diagnostics panel runs entirely locally without AI calls.

Can I check my usage without making a request?

Yes, the Settings panel shows cached usage information. The data syncs with the server periodically and after each request.

What if I need unlimited for a day then go back to free?

You can switch between modes freely. Use your API key for intensive sessions, then disable it to return to server proxy mode.
