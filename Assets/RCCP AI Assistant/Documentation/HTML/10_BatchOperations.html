RCCP AI Assistant supports batch operations for creating or customizing multiple vehicles at once. This feature is ideal for setting up vehicle fleets, applying consistent configurations across variants, or rapidly prototyping multiple vehicle types.

Overview

Batch operations allow you to:

  • Select multiple raw 3D models and create RCCP vehicles for all of them
  • Select multiple existing RCCP vehicles and apply the same customization to all
  • Use a single description that applies to all selected items
  • Process vehicles sequentially with a single AI generation

Batch Vehicle Creation

When to Use

Batch creation is useful when:

  • You have multiple car models that need similar configurations
  • You're setting up a vehicle lineup for a racing game
  • You want to prototype several vehicles quickly
  • You need consistent base configurations across a fleet

How to Select Multiple Models

  1. In the Hierarchy window, hold Ctrl (Windows) or Cmd (Mac)
  2. Click each 3D model root you want to include
  3. Or use Shift+Click to select a range

The AI Assistant shows:

  • "X models selected" indicator
  • List of selected model names
  • Combined eligibility summary

Requirements for Batch Creation

Each selected model must:

  • Be a raw 3D model (no existing RCCP_CarController)
  • Be in the scene (not project assets)
  • Pass basic eligibility checks (scale, wheels)

Models that fail eligibility are flagged but don't block the operation.

Writing a Batch Description

Write a single description that applies to all vehicles:

Good batch descriptions: 


Create standard passenger sedans with 180hp front-wheel drive,

automatic transmission, and balanced handling for daily driving.

All vehicles should have moderate stability control.



Set up race cars with 350hp rear-wheel drive, manual transmission,

stiff suspension, and minimal stability assists for track use.

Consider that:

  • The description applies identically to all vehicles
  • Model-specific characteristics (size, shape) aren't considered
  • You can customize individual vehicles afterward

Processing Flow

  1. Click Generate with multiple models selected
  2. The AI generates a single configuration
  3. Click Apply to create all vehicles
  4. Each model is processed sequentially:
  • RCCP components added
  • Configuration applied
  • Progress shown
  1. Post-creation wizard appears for each vehicle (optional)

Progress Display

During batch processing: 


Processing: 3 of 8 vehicles

Current: SportsCar_Model_03

Status: Adding components...

A progress bar shows overall completion.

Handling Failures

If a vehicle fails during batch creation:

  • An error message is logged
  • Processing continues to the next vehicle
  • A summary shows successes and failures
  • Failed vehicles can be retried individually

Batch Vehicle Customization

When to Use

Batch customization is useful when:

  • You need to apply the same tuning to multiple vehicles
  • You're balancing a vehicle roster
  • You want consistent changes across variants
  • You're updating a fleet configuration

How to Select Multiple RCCP Vehicles

  1. Select multiple GameObjects that have RCCP_CarController
  2. The assistant automatically identifies RCCP vehicles
  3. Duplicates (selecting both a parent and child) are deduplicated

Note: If you select a mix of RCCP vehicles and raw models, only the RCCP vehicles are included for customization.

Writing a Batch Customization Prompt

Your description applies to all selected vehicles: 


Enable ABS on all vehicles with moderate intervention.

Add light TCS for wet conditions.



Increase engine power by 20% across all vehicles.

Stiffen suspension for better track handling.

Processing Flow

  1. A single AI generation creates the configuration
  2. The same changes apply to each vehicle
  3. Before/after state is captured per-vehicle
  4. Changes are applied sequentially
  5. Summary shows results for all vehicles

Considerations for Batch Customization

Percentage vs. Absolute Changes

  • "Increase power by 20%" scales with each vehicle's current power
  • "Set power to 300hp" makes all vehicles identical

Vehicle Differences

Each vehicle's current state is captured independently, so percentage-based changes produce proportionally different results.

Undo Behavior

Batch customization creates grouped undo entries. A single Undo reverts all vehicles.

Batch Limitations

Available Panels

Batch operations are only available in:

  • Vehicle Creation - Multiple raw models
  • Vehicle Customization - Multiple RCCP vehicles

Other panels (Behaviors, Wheels, Audio, Lights, Damage, Diagnostics) require single selection. This ensures focused, component-specific configuration.

Selection Limits

There's no hard limit on batch size, but consider:

  • Processing time increases linearly
  • Editor may slow with many complex operations
  • 8-12 vehicles is a practical maximum for comfortable workflow

Post-Creation Handling

For batch creation:

  • Wheel assignment wizard appears for the first vehicle
  • Option to skip/apply same assignments to remaining vehicles
  • Body colliders can be added per-vehicle or skipped

Best Practices

Prepare Models First

Before batch creation:

  1. Ensure all models meet requirements
  2. Standardize naming conventions
  3. Check scale and orientation
  4. Organize in hierarchy for easy selection

Use Consistent Naming

Name your models predictably: 


Vehicle_Sedan_01

Vehicle_Sedan_02

Vehicle_SUV_01

Vehicle_SUV_02

This makes selection and tracking easier.

Test with Small Batches

Before processing many vehicles:

  1. Test with 2-3 vehicles first
  2. Verify results match expectations
  3. Adjust description if needed
  4. Then process the full batch

Document Your Batch Prompts

Save successful batch descriptions for:

  • Reproducing configurations later
  • Sharing with team members
  • Creating documentation

Plan for Individual Tuning

Batch operations create a baseline. Plan to:

  • Customize individual vehicles afterward
  • Use specialized panels for vehicle-specific tuning
  • Iterate based on playtesting

Use Cases

Racing Game Fleet

Scenario: Create 12 different race cars with consistent performance.

  1. Import 12 car models
  2. Select all in Hierarchy
  3. Batch create with:

```

High-performance race cars with 400hp engines, rear-wheel drive,

sequential gearbox, stiff racing suspension, and minimal assists.

```

  1. Individual tune each car for variety

City Traffic Vehicles

Scenario: Populate a city scene with diverse traffic.

  1. Select 8 civilian vehicle models
  2. Batch create with:

```

Standard civilian vehicles with moderate power, automatic transmission,

front-wheel drive, comfortable suspension, and full stability systems.

```

  1. The AI NPC system can use these consistently-configured vehicles

Vehicle Variants

Scenario: Create sport, touring, and economy variants of one car.

  1. Duplicate the base model 3 times
  2. Batch create with base config
  3. Individually customize each for its purpose:
  • Sport: More power, stiffer suspension
  • Touring: Comfort-focused, quiet
  • Economy: Lower power, efficient

Rebalancing Update

Scenario: Adjust all player vehicles after testing feedback.

  1. Select all RCCP vehicles in the game
  2. Batch customize:

```

Reduce overall grip by 10% for more challenging handling.

Increase ABS intervention slightly for better brake feel.

```

  1. All vehicles get consistent adjustments

Troubleshooting

"No valid models selected"

Ensure your selection includes:

  • Scene objects (not Project assets)
  • Root GameObjects (not child components)
  • Raw models (for creation) or RCCP vehicles (for customization)

Some vehicles not processing

Check individual vehicles for:

  • Disabled GameObjects
  • Missing required components
  • Eligibility failures

Inconsistent results

Batch operations apply identical configuration. Different results usually mean:

  • Vehicle geometry differences affect physics
  • Existing configurations varied before batch customization

Editor slowdown

For large batches:

  • Close unnecessary editor windows
  • Ensure adequate system resources
  • Consider processing in smaller batches
